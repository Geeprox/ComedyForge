# Persona + Audience Generator Protocol

## 1. 输入参数
- `theme`：创作主题（必填）
- `batch-size`：10-500
- `audience-size`：10-200
- `round`：当前轮次（从1开始）
- `history`：上一轮保留段子与观众摘要（不含原始打分细节）

## 2. 创作者人格生成（两层结构）

### 2.1 专业基线（常量，必须注入每个Persona）
所有段子人格必须同时满足：
1. 深爱喜剧行业，尊重观众，敬畏演出机会，尊重作品。
2. 5年以上喜剧创作与开放麦实战经验。
3. 深度研读《李诞脱口秀工作手册》，并内化方法论：说真话、建立预期、打破预期、结构收束、callback回收。
4. 融会贯通并避免机械模仿：
   - 李诞：松弛感 + 结构控制
   - 呼兰：高密度文字游戏 + 知识性
   - 庞博：真诚叙事 + callback设计
   - 徐志胜：缺陷美学 + 角色塑造
   - 鑫仔：漫才节奏 + 双人气场
   - 周奇墨：观察深度 + 细节刻画
   - 鸟鸟：内向视角 + 精准描述
   - 童漠男：愤怒管理 + 荒诞推演

### 2.2 多样性变量（8维，采用“总库 + 激活子集”采样）
1. 身份处境（长期结构角色）：
   `实习生/中层/高管/外包/自由职业/个体户/全职家长/待业求职/创业失败回流/体制内边缘岗/跨城通勤族/退休返聘`
2. 代际背景：
   `GenZ/Zalpha/Millennials/GenX/Silver/夹心代`
3. 性格基调：
   `乐观/悲观/愤世/麻木/亢奋/社恐/讨好型/控制欲/好胜/拧巴/爱抬杠`
4. 认知滤镜（规范化标签）：
   `经济学/行为心理/法律合规/医学健康/产品经理/工程师/人力资源/互联网运营/消费者维权/家庭伦理/教育/历史/体育/投资理财/玄学/游戏/科幻/动物拟人`
5. 语言风格：
   `学术/互联网黑话/方言/诗意/机械直白/冷面短句/碎碎念/播客口语/客服话术/官腔`
6. 隐藏秘密（长期隐患）：
   `瞒家人被裁/副业亏损/信用卡逾期/偷偷备考转行/准备离职创业/婚礼预算爆炸/家里催生压力/背着对象还债/暗恋同事/租房即将断供`
7. 当下状态（短时事件）：
   `宿醉未醒/刚分手/刚和好/刚离婚/刚结婚/刚失业/刚升职/刚被HR约谈/刚体检异常/刚搬家/租约到期被催搬/准备求婚`
8. 与主题关系：
   `受害者/加害者/旁观者/拯救者/逃离者/既得利益者/共谋者/误伤者/审判者`

维度边界（强制）：
- `刚离婚/刚结婚/刚失业`等短时事件必须放在“当下状态”，不得放入“身份处境”。
- “身份处境”只允许长期结构角色，不得混入一次性事件标签。

### 2.3 采样规则（避免“枚举越多越稀释”）
先“激活子集”再“逐段子采样”：
1. 每轮先为每个维度从总库抽取激活值集合`active_values(dim)`。
2. 该轮段子仅在`active_values(dim)`内采样，不在全量库直接等概率乱抽。
3. 轮末复检覆盖率与单值样本数，不达标则重采样。

激活子集规模（按`batch-size`自适应，受各维总库上限约束）：
- 10-29：每维激活4-6个值
- 30-79：每维激活6-10个值
- 80-200：每维激活8-14个值
- 201-500：每维激活10-16个值

单值样本数护栏：
```text
expected_count_per_value = batch_size / active_count(dim)
batch_size < 30          : expected_count_per_value in [2, 6]
30 <= batch_size < 80    : expected_count_per_value in [3, 10]
80 <= batch_size <= 200  : expected_count_per_value in [4, 18]
200 < batch_size <= 500  : expected_count_per_value >= 8（不设硬上限，受维度总库约束）
若低于下限 -> 缩小active_count(dim)
若高于上限（仅前三档） -> 扩大active_count(dim)
```

批次覆盖底线：
- 单个Batch中身份处境至少覆盖4种（`batch-size >= 50`时至少5种）。
- 认知滤镜至少覆盖5种（`batch-size >= 50`时至少6种）。
- 当下状态至少覆盖4种（`batch-size >= 50`时至少5种）。
- 语言风格至少覆盖3种（`batch-size >= 50`时至少4种）。
- 单一代际占比控制在20%-50%。
- 相邻段子语言风格必须不同。
- 新血段子在“认知滤镜”与旧精英集合的差异度必须大于0.5。

组合一致性约束：
- 同轮中不同段子不得复用同一“隐藏秘密 + 当下状态”组合。
- “隐藏秘密”与“当下状态”不得语义同义重复（如两者都表示“刚失业”）。
- 认知滤镜必须做标签归一化后再参与去重与差异度计算（如`法学/律师视角 -> 法律合规`）。

### 2.4 默认采样权重（Core/Long-tail）
目标：提高稳定性，同时保留探索空间，避免“全量均匀抽样”。

权重分层（建议默认）：
- Core值总权重：70%-80%
- Long-tail值总权重：20%-30%

执行流程：
1. 先在每个维度标记`core_values(dim)`与`long_tail_values(dim)`。
2. 生成`active_values(dim)`时，优先满足`core_quota = ceil(active_count * 0.6)`，剩余槽位从long-tail补齐。
3. 段子级采样时在`active_values(dim)`内按权重抽样，而非等概率抽样。

段子级采样公式（建议）：
```text
p(v | dim) ∝ exp(
  log(base_weight(v))
  + novelty_bonus(v)
  - repetition_penalty(v)
  + theme_affinity(v, theme)
)
```

建议取值：
- `novelty_bonus`：当轮低频值 `+0.10 ~ +0.25`
- `repetition_penalty`：当轮高频值 `0.10 ~ 0.30`
- `theme_affinity`：命中主题语义时 `+0.05 ~ +0.20`

### 2.5 跨轮动态调权（防模式坍塌）
每轮结束后，按分布体检与得分反馈更新下一轮权重：
1. 若某维`normalized_entropy < 0.72`：提升该维long-tail总权重`+0.05`（上限0.35）。
2. 若某值连续2轮占比进入Top1且得分无显著优势（`delta < 0.05`）：对该值加`repetition_penalty +0.10`。
3. 若某值属于外卡/精英且提升显著（`delta >= 0.20`）：下一轮保留轻度加权`+0.05`，仅持续1轮。
4. 若“隐藏秘密 + 当下状态”组合重复率超阈值：优先调整“当下状态”权重，不先改“身份处境”。

权重安全阈值：
- 单个值权重上限：`0.22`
- 单个值权重下限：`0.03`
- 任一维long-tail总权重上限：`0.35`

差异度计算：
```text
distance(new_filter, elite_filters) = 1 - max(sim(new_filter, e) for e in elite_filters)
sim = 语义映射后Jaccard或余弦相似度
要求 distance > 0.5
```

## 3. 观众生成（12维特征向量）

### 3.1 维度定义
- 极高权重（每类强制配额≥15%）：`代际`、`职业领域`、`喜剧阅历`
- 高权重（每类强制配额≥10%）：`性别`、`性格倾向`、`地域文化`、`禁忌话题`
- 中权重（保证存在性）：`职级`、`当日遭遇`、`文化倾向`、`当日心情`
- 噪声：纯随机扰动

### 3.2 相关性采样（避免不可能组合）
- `GenZ <-> Student` 相关性 +0.8
- `暴躁心情 <-> 被裁遭遇` 相关性 +0.7
- `同行阅历 <-> 批判性格` 相关性 +0.6

实现建议：
```text
先采样主维度，再用条件概率重采样强相关维度；
若组合落入低可行区间（概率阈值<0.1），则回退并重采样。
```

### 3.3 场次差异（Atmosphere Seed）
- `seed = hash(theme + round + timestamp_bucket)`
- 用seed控制本轮观众宏观氛围（宽容/挑剔/疲惫/亢奋等）
- 与上一轮观众画像做Jaccard比较，要求相似度 `< 0.3`

Seed必须显式影响观众采样分布，不可只做日志字段：
- 默认情绪采样基线：`疲惫/暴躁`合计占比40%，其余情绪占比60%。
- 当seed语义命中“雨夜沮丧/加班崩溃/地铁晚点”等负向氛围时：`疲惫/暴躁`合计占比提升到60%。
- 当seed语义命中“节日庆典/发薪日/团建放假”等正向氛围时：`疲惫/暴躁`合计占比下调到25%-30%。

建议实现：
```text
default_mood_weights = {疲惫:0.22, 暴躁:0.18, 乐观:0.22, 平静:0.20, 亢奋:0.18}

if seed in NEGATIVE_ATMOSPHERE:
  mood_weights = rebalance(default_mood_weights, target_sum(疲惫+暴躁)=0.60)
elif seed in POSITIVE_ATMOSPHERE:
  mood_weights = rebalance(default_mood_weights, target_sum(疲惫+暴躁)=0.25~0.30)
else:
  mood_weights = default_mood_weights
```

评分松紧联动（在Audience Simulator执行）：
- `疲惫`观众：更保守，平均打分偏移 `-0.15`
- `暴躁`观众：更严苛，平均打分偏移 `-0.25`
- `乐观`观众：更宽松，平均打分偏移 `+0.10`
- 偏移后再裁剪到1-5星整数区间。

### 3.4 Audience > 100 自动亚组
固定2x2矩阵：
- A组(25%)：同行 + 批判
- B组(25%)：同行 + 乐观
- C组(25%)：新手 + 批判
- D组(25%)：新手 + 乐观

## 4. 输出对象结构
创作者输出Joke草案（含persona标签），评审侧输出Audience向量（无创作者可见打分字段）。

## 5. 隔离规则
- 创作者只接收“上一轮摘要”和“进化方向”，不可见评审个体评分。
- 评审团不可见创作者隐藏人格字段与生成规则权重。
